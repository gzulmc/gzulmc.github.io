    LangChain流式传输摘要

    基于您提供的LangChain文档，我为您系统性地归纳了流式输出（Streaming）的核心知识点与关键注意事项。

    核心知识点

    1. 目的与价值
    2. 提升用户体验：通过逐步、实时地显示输出（即使在完整响应就绪前），有效应对LLM的延迟问题，是构建响应式应用的关键。
    3. 三种基础流式模式 (stream_mode) 您可以通过 stream 或 astream 方法的 stream_mode 参数指定一种或多种模式：
    4. "updates" (智能体进度)：在每个智能体步骤（如LLM调用、工具执行）后，流式传输状态更新。适合展示智能体的工作流程。
    5. "messages" (LLM令牌)：流式传输来自任何LLM调用的令牌元组 (token, metadata)。这是实现“打字机”效果和获取工具调用块的基础。
    6. "custom" (自定义更新)：在图形节点内部使用 get_stream_writer() 函数，流式传输任意用户定义的数据（如进度提示），适合发送应用层信号。
    7. 常见高级模式
    8. 流式推理令牌：使用 stream_mode="messages"，并过滤内容块（content_blocks）中类型为 "reasoning" 的块。LangChain会将不同提供商的推理格式统一为标准格式。
    9. 流式工具调用：可以流式传输工具调用的部分JSON参数，同时通过组合 stream_mode=["messages", "updates"] 来获取完整、解析后的工具调用和工具执行结果。
    10. 多人机回环：结合 HumanInTheLoopMiddleware 和检查点（checkpointer），可在流式过程中捕获中断（__interrupt__），收集用户决策（批准/编辑），并通过 Command 恢复执行流。
    11. 子智能体流式：为子智能体设置 name 参数。在流式传输时，可通过元数据中的 lc_agent_name 键区分事件来源，并使用 subgraphs=True 获取嵌套流。
    12. 推荐格式：v2 流式格式
    13. 强烈建议在 stream() 中传入 version="v2"。
    14. 它会返回统一的 StreamPart 字典（包含 type, ns, data 键），使多模式流式的处理更清晰、一致。

    重要注意事项

    1. 推荐使用事件流式传输
    2. 文档明确指出，对于新应用，推荐使用事件流式传输 (Event Streaming)（即 stream_events(..., version="v3") 接口，您提供的另一份文档有详细说明）。它提供了类型化的投影（如 .messages, .tool_calls），可以独立消费不同数据流，比手动解析 stream_mode 块更优雅。
    3. 生产环境配置
    4. 持久化历史：在生产环境中，必须通过 config 传递 thread_id 并配置检查点（checkpointer，如 InMemorySaver）来持久化对话历史，以便后续轮次恢复。在LangSmith部署中会自动配置。
    5. 功能限制与兼容性
    6. 自定义流的上下文限制：如果在工具内部使用 get_stream_writer()，该工具将无法在LangGraph执行上下文之外被调用。
    7. 禁用流式：可通过在模型初始化时设置 streaming=False 来禁用令牌级流式（对不支持该参数的模型，使用 disable_streaming=True）。
    8. 模型支持：并非所有模型集成都支持 streaming 参数，且推理令牌功能需要在模型端单独启用。
    9. 数据访问与聚合技巧
    10. 访问完整消息：当完整消息不直接在状态更新中时，可以通过自定义更新发送，或在流式循环中手动聚合 AIMessageChunk 块（检查 chunk_position == "last"）。
    11. 标准化内容块：通过 token.content_blocks 属性可以统一访问不同提供商的各种内容类型（文本、推理、工具调用等）。

    总结

    LangChain的流式传输核心是三种基础模式，并在此基础上构建了推理展示、工具调用监控、人机回环等高级模式。核心建议是：

    • 新项目优先采用 stream_events(version="v3") 以获得更清晰的架构。
    • 若使用底层 stream()，务必使用 version="v2" 并明确所需的 stream_mode。
    • 生产环境务必配置好 thread_id 和检查点。
